Inheritance in Java

1. Declaring a Subclass

In Java, you use the extends keyword to declare that a class is a subclass of another class. The subclass inherits the fields and methods of the superclass. Here's a simple example:

    
    class Animal {
        void eat() {
            System.out.println("Animal is eating");
        }
    }
    
    class Dog extends Animal {
        void bark() {
            System.out.println("Dog is barking");
        } class="heading-3"
    }

2. Accessing Superclass Members

The super keyword is used to refer to members (fields or methods) of the superclass within the subclass. This is useful when you want to access or override a method from the superclass.

    
    class Animal {
        void eat() {
            System.out.println("Animal is eating");
        }
    }
    
    class Dog extends Animal {
        void eat() {
            super.eat(); // Call the eat() method of the superclass
            System.out.println("Dog is eating");
        }
    }

3. Constructor Chaining

Constructors in a subclass can call constructors of the superclass using super(). This allows you to initialize both the subclass and superclass parts of an object.

    
    class Animal {
        Animal() {
            System.out.println("Animal constructor");
        }
    }
    
    class Dog extends Animal {
        Dog() {
            super(); // Call the constructor of the superclass
            System.out.println("Dog constructor");
        }
    }

4. Types of Inheritance

Java supports various types of inheritance including single, multiple (through interfaces), multilevel, hierarchical, and hybrid inheritance.

    
    // Example of Multiple Inheritance through Interfaces
    interface Walkable {
        void walk();
    }
    
    interface Swimmable {
        void swim();
    }
    
    class Amphibian implements Walkable, Swimmable {
        public void walk() {
            // ...
        }
    
        public void swim() {
            // ...
        }
    }

5. Method Overriding

Subclasses can provide their own implementation of a method that is already defined in the superclass. This is called method overriding.

    
    class Animal {
        void makeSound() {
            System.out.println("Animal makes a sound");
        }
    }
    
    class Dog extends Animal {
        void makeSound() {
            System.out.println("Dog barks");
        }
    }

6. Final and Inheritance

You can use the final keyword to prevent a class or method from being extended or overridden.

    
    final class FinalClass {
        // ...
    }
    
    class SubClass /*extends FinalClass*/ { // This would result in a compilation error
        // ...
    }

7. Abstract Classes and Inheritance

Abstract classes in Java are often used as base classes that are meant to be extended by concrete subclasses. Abstract classes can have abstract methods that must be implemented by concrete subclasses.

    
    abstract class Shape {
        abstract void draw(); // Abstract method
    }
    
    class Circle extends Shape {
        void draw() {
            System.out.println("Drawing a circle");
        }
    }

8. Object Class and Inheritance

In Java, every class implicitly extends the Object class. The Object class provides common methods like toString, equals, and hashCode. This means that any class you create is a subclass of Object.

    
    class MyClass {
        // ...
    }
    
    // Equivalent to: class MyClass extends Object

Conclusion

Inheritance is a powerful feature in Java that promotes code reuse and helps in creating well-organized, modular, and extensible software. It allows for the creation of class hierarchies that model real-world relationships and behaviors. Understanding inheritance is crucial for effective object-oriented design and programming in Java.


Types of Inheritance in Java

1. Single Inheritance

Single inheritance refers to the scenario where a class can inherit from only one superclass. In other words, a subclass extends a single superclass. 

    
    class Animal {
        // ...
    }

    class Dog extends Animal {
        // ...
    }

2. Multiple Inheritance (through Interfaces)

Java supports multiple inheritance through interfaces. A class can implement multiple interfaces, allowing it to inherit the abstract methods defined in each interface. 

    
    interface Walkable {
        void walk();
    }

    interface Swimmable {
        void swim();
    }

    class Amphibian implements Walkable, Swimmable {
        public void walk() {
            // ...
        }

        public void swim() {
            // ...
        }
    }

3. Multilevel Inheritance

In multilevel inheritance, a class extends another class, and then another class extends that derived class, forming a chain of inheritance. 

    
    class Animal {
        // ...
    }

    class Mammal extends Animal {
        // ...
    }

    class Dog extends Mammal {
        // ...
    }

4. Hierarchical Inheritance

Hierarchical inheritance occurs when multiple classes inherit from a single superclass. It forms a tree-like structure. 

    
    class Shape {
        // ...
    }

    class Circle extends Shape {
        // ...
    }

    class Rectangle extends Shape {
        // ...
    }

5. Hybrid Inheritance

Hybrid inheritance is a combination of two or more types of inheritance. It often involves a mix of single, multiple, multilevel, or hierarchical inheritance. 

    
    class A {
        // ...
    }

    class B extends A {
        // ...
    }

    interface C {
        // ...
    }

    class D extends B implements C {
        // ...
    }

It's important to note that Java does not support multiple inheritance through classes to avoid the "diamond problem." The "diamond problem" arises when a class inherits from two classes that have a common ancestor, leading to ambiguity in method resolution. By supporting multiple inheritance through interfaces, Java avoids this issue while still allowing flexibility in class design.

Problem 1: Animal Hierarchy

Create a class Animal with properties like name and sound. Derive two subclasses, Mammal and Bird, each with specific properties. Demonstrate single inheritance hierarchy. 


class Animal {
    String name;
    String sound;

Create interfaces Shape and Color with methods related to shapes and colors. Implement these interfaces in a class ColoredRectangle to demonstrate multiple inheritance through interfaces. 


interface Shape {
    double calculateArea();
}

interface Color {
    String getColor();
}

class ColoredRectangle implements Shape, Color {
    int width;
    int height;
    String color;

    ColoredRectangle(int width, int height, String color) {
        this.width = width;
        this.height = height;
        this.color = color;

    public double calculateArea() {

Hybrid Inheritance Examples:


Problem 9: Using Single and Multiple Inheritance

This example demonstrates hybrid inheritance in Java using a combination of single and multiple inheritance. The base class Animal has an eat method. The Dog class is a subclass of Animal with an additional bark method. Another base class Bird has a fly method. The Parrot class is a subclass of Bird with an additional talk method. The DogParrot class is a subclass of Dog and has its own talkAndFly method. In the main method, an instance of DogParrot is created and methods from the different classes are called. 


class Animal {
    void eat() {
        System.out.println("Animal is eating");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog is barking");
    }
}

class Bird {
    void fly() {
        System.out.println("Bird is flying");
    }
}

class Parrot extends Bird {
    void talk() {
        System.out.println("Parrot is talking");
    }
}

class DogParrot extends Dog {
    void talkAndFly() {
        System.out.println("DogParrot is talking and flying");
    }
}

public class HybridInheritanceExample {
    public static void main(String[] args) {
        DogParrot dp = new DogParrot();
        dp.eat();      // Inherited from Animal
        dp.bark();     // Inherited from Dog
        dp.talkAndFly(); // Own method in DogParrot
    }
}

Problem 10: Using Interface and Multiple Inheritance

This example demonstrates hybrid inheritance using an interface and multiple inheritance. The Swimmer interface has a swim method. The base class Animal has an eat method. The Fish class is a subclass of Animal and implements Swimmer. Another base class Bird has a fly method. The FlyingFish class is a subclass of Fish and has its own flyAndSwim method. In the main method, an instance of FlyingFish is created, and methods from the different classes and interfaces are called. 


interface Swimmer {
    void swim();
}

class Animal {
    void eat() {
        System.out.println("Animal is eating");
    }
}

class Fish extends Animal implements Swimmer {
    void swim() {
        System.out.println("Fish is swimming");
    }
}

class Bird {
    void fly() {
        System.out.println("Bird is flying");
    }
}

class FlyingFish extends Fish {
    void flyAndSwim() {
        System.out.println("FlyingFish is flying and swimming");
    }
}

public class HybridInheritanceExample2 {
    public static void main(String[] args) {
        FlyingFish ff = new FlyingFish();
        ff.eat();         // Inherited from Animal
        ff.swim();        // Inherited from Swimmer
        ff.flyAndSwim();  // Own method in FlyingFish
    }
}
