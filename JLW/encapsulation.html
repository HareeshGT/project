Encapsulation in Java

1. Private Access Modifier:

In Java, the private access modifier is used to restrict the visibility of a field or method to only within its own class. This ensures that the member is not accessible from outside the class, promoting encapsulation.

Using private for fields: 


        public class Person {
            private String name;
            private int age;
            // ...
        }

2. Getter and Setter Methods:

Getter and setter methods are used to provide controlled access to the private fields of a class. Getter methods retrieve the values of fields, and setter methods modify the values, enforcing encapsulation rules.

Example with getter and setter methods: 


        public class Person {
            private String name;
            private int age;
        
            // Getter method for name
            public String getName() {
                return name;
            }
        
            // Setter method for name
            public void setName(String name) {
                this.name = name;
            }
        
            // Getter method for age
            public int getAge() {
                return age;
            }
        
            // Setter method for age with validation
            public void setAge(int age) {
                if (age > 0) {
                    this.age = age;
                } else {
                    System.out.println("Invalid age");
                }
            }
        }

3. Benefits of Encapsulation:

  • Controlled Access: Encapsulation ensures that the internal details of an object are not directly accessible.
  • Flexibility and Maintenance: Changes to the internal implementation details do not affect external code, promoting flexibility.
  • Security: Prevents unauthorized access and manipulation of the internal state.
  • Code Organization: Groups related data and behavior within a class for improved readability.
  • Facilitates Reusability: Provides well-defined interfaces for interacting with objects.

4. Encapsulation Guidelines:

  • Minimize Accessibility: Make fields as private as possible to limit direct access.
  • Provide Controlled Access: Use getter and setter methods to control access to private fields.
  • Immutable Classes: Consider making classes immutable by not providing setter methods.
  • Consistent Naming Conventions: Follow consistent naming conventions for getter and setter methods.
  • Avoid Public Fields: Avoid using public fields directly; always encapsulate fields.

Access Specifiers in Encapsulation (Java)

1. Public (`public`):

The public access specifier allows a class, method, or field to be accessed from any other class in the same or different package. 

                
        public class Example {
            public int publicField;
            public void publicMethod() {
                // method implementation
            }
        }

2. Private (`private`):

The private access specifier restricts the visibility of a class, method, or field to only within the same class. It is the most restrictive access level. 

                
        public class Example {
            private int privateField;
            private void privateMethod() {
                // method implementation
            }
        }

3. Protected (`protected`):

The protected access specifier allows a class, method, or field to be accessed within the same package or by a subclass (even if it is in a different package). 

                
        public class Example {
            protected int protectedField;
            protected void protectedMethod() {
                // method implementation
            }
        }

4. Default (Package-Private):

If no access specifier is specified, it defaults to package-private or "default" access. Members with default access are visible only within the same package. 

                
        class Example {
            int defaultField; // package-private
            void defaultMethod() {
                // method implementation
            }
        }

Encapsulation and Access Specifiers:

  • Private Access: Fields are often declared as private to encapsulate them, preventing direct access from outside the class.
  • Public Access for Relevant Methods: Methods that are part of the public interface of a class may have a public access specifier to allow external classes to use them.
  • Protecting Internal State: The internal state of a class is often protected by using private access for fields and providing controlled access through public or protected methods.

Explanation

Encapsulation is a fundamental principle in object-oriented programming that involves bundling data (attributes) and methods (functions) into a single unit known as a class. The example above demonstrates encapsulation in Java through the creation of a `Person` class.

Benefits of Encapsulation

  • Controlled Access: Encapsulation ensures that the internal details of an object are not directly accessible. Access is controlled through well-defined interfaces (getter and setter methods).
  • Flexibility and Maintenance: Changes to the internal implementation details of a class do not affect the external code that uses the class. This promotes flexibility and eases maintenance.
  • Security: Encapsulation contributes to security by preventing unauthorized access and manipulation of the internal state of an object.
  • Code Organization: Encapsulation organizes code by grouping related data and behavior within a class. This improves code readability and maintainability.
  • Facilitates Reusability: Encapsulation promotes reusability by providing well-defined interfaces for interacting with objects. Classes with encapsulated functionality can be easily reused in different contexts.

Example 1: Basic Encapsulation

The following Java program demonstrates basic encapsulation with a Student class. Private fields (name and age) are encapsulated using getter and setter methods. 

            
    public class Student {
        private String name;
        private int age;
    
        // Constructor
        public Student(String name, int age) {
            this.name = name;
            this.age = age;
        }
    
        // Getter method for name
        public String getName() {
            return name;
        }
    
        // Setter method for name
        public void setName(String name) {
            this.name = name;
        }
    
        // Getter method for age
        public int getAge() {
            return age;
        }
    
        // Setter method for age with validation
        public void setAge(int age) {
            if (age > 0) {
                this.age = age;
            } else {
                System.out.println("Invalid age");
            }
        }
    
        public static void main(String[] args) {
            // Creating a Student object
            Student student = new Student("John", 20);
    
            // Accessing private fields using getter methods
            System.out.println("Name: " + student.getName());
            System.out.println("Age: " + student.getAge());
    
            // Modifying private fields using setter methods
            student.setName("Jane");
            student.setAge(21);

            System.out.println("Updated Name: " + student.getName());
            System.out.println("Updated Age: " + student.getAge());

    }

Example 4: Encapsulation with Complex Data Type and Validation

The Library class encapsulates a list of Book objects. It includes getter and setter methods for the library name, a getter method for the list of books (returning an unmodifiable view), and a method to add books with validation. 

            
    public class Library {
        private String name;
        private List<Book> books;
    
        // Constructor
        public Library(String name) {
            this.name = name;
            this.books = new ArrayList<>();
        }
    
        // Getter method for name
        public String getName() {
            return name;
        }
    
        // Setter method for name
        public void setName(String name) {
            this.name = name;
        }
    
        // Getter method for books
        public List<Book> getBooks() {
            return Collections.unmodifiableList(books);
        }
    
        // Method to add a book to the library
        public void addBook(Book book) {
            if (book != null) {
                books.add(book);
            }
        }
    
        public static void main(String[] args) {
            // Creating a Library object
            Library library = new Library("City Library");
    
            // Creating Book objects
            Book book1 = new Book("The Catcher in the Rye", "J.D. Salinger");
            Book book2 = new Book("To Kill a Mockingbird", "Harper Lee");
    
            // Adding books to the library
            library.addBook(book1);
            library.addBook(book2);
    
            // Accessing private fields using getter methods
            System.out.println("Library Name: " + library.getName());
            System.out.println("Books in the Library: " + library.getBooks());
        }
    }
    
    class Book {
        private String title;
        private String author;
    
        // Constructor
        public Book(String title, String author) {
            this.title = title;
            this.author = author;
        }
    
        // Getter method for title
        public String getTitle() {
            return title;
        }
    
        // Getter method for author
        public String getAuthor() {
            return author;
        }
    
        @Override
        public String toString() {
            return title + " by " + author;
        }
    }

Example 5: Encapsulation with Modifiers

The Employee class demonstrates encapsulation with modifiers. It encapsulates the employee's name and salary. Getter and setter methods provide access to private fields, and a method is included to calculate the employee's bonus. 

            
    public class Employee {
        private String name;
        private double salary;
        private static final double BONUS_PERCENTAGE = 0.1;
    
        // Constructor
        public Employee(String name, double salary) {
            this.name = name;
            this.salary = salary;
        }
    
        // Getter method for name
        public String getName() {
            return name;
        }
    
        // Setter method for name
        public void setName(String name) {
            this.name = name;
        }
    
        // Getter method for salary
        public double getSalary() {
            return salary;
        }
    
        // Method to calculate bonus
        public double calculateBonus() {
            return salary * BONUS_PERCENTAGE;
        }
    
        public static void main(String[] args) {
            // Creating an Employee object
            Employee employee = new Employee("Alice", 50000);
    
            // Accessing private fields using getter methods
            System.out.println("Employee Name: " + employee.getName());
            System.out.println("Employee Salary: $" + employee.getSalary());
    
            // Calculating and displaying bonus
            System.out.println("Bonus: $" + employee.calculateBonus());
        }
    }

Example 6: Encapsulation in a Banking System

The BankAccount class encapsulates the account number and balance in a simple banking system. Getter methods provide access to private fields, and methods are included for deposit and withdrawal, each with validation checks. 

            
    public class BankAccount {
        private String accountNumber;
        private double balance;
    
        // Constructor
        public BankAccount(String accountNumber, double initialBalance) {
            this.accountNumber = accountNumber;
            this.balance = initialBalance;
        }
    
        // Getter method for account number
        public String getAccountNumber() {
            return accountNumber;
        }
    
        // Getter method for balance
        public double getBalance() {
            return balance;
        }
    
        // Method to deposit money
        public void deposit(double amount) {
            if (amount > 0) {
                balance += amount;
                System.out.println("Deposit successful. New balance: $" + balance);
            } else {
                System.out.println("Invalid deposit amount");
            }
        }
    
        // Method to withdraw money
        public void withdraw(double amount) {
            if (amount > 0 && amount <= balance) {
                balance -= amount;
                System.out.println("Withdrawal successful. New balance: $" + balance);
            } else {
                System.out.println("Invalid withdrawal amount or insufficient funds");
            }
        }
    
        public static void main(String[] args) {
            // Creating a BankAccount object
            BankAccount account = new BankAccount("123456", 1000);
    
            // Accessing private fields using getter methods
            System.out.println("Account Number: " + account.getAccountNumber());
            System.out.println("Initial Balance: $" + account.getBalance());
    
            // Performing transactions
            account.deposit(500);
            account.withdraw(200);
        }
    }
