Java Classes and Objects

Class in Java:

  1. Definition:

    A class is a blueprint or a template for creating objects. It defines the properties (fields) and behaviors (methods) that objects created from the class will have.

  2. Key Components:

    This section lists the important components of a class, including fields, methods, constructors, and modifiers.

    • Fields (Data Members):

      These are variables that represent the attributes or properties of the class. Fields store data specific to each object created from the class.

    • Methods (Member Functions):

      These are functions that define the behaviors or actions that objects of the class can perform. Methods operate on the fields to achieve specific functionalities.

    • Constructors:

      Special methods used for initializing objects when they are created. Constructors have the same name as the class and are called automatically when an object is instantiated.

    • Modifiers:

      Keywords like public, private, and protected that control the access level of the members (fields and methods) of a class.

  3. Example Class:

    This section provides an example class named "Car" with fields, constructor, and a method. It illustrates how a class is structured in Java.



Object in Java:

  1. Definition:

    An object is an instance of a class. It represents a real-world entity and is created from a class definition. Objects have state (defined by the class's fields) and behavior (defined by the class's methods).

  2. Creating Objects:

    This section explains how to create objects of a class using the new keyword and illustrates how objects are instances of a class.

  3. Example Object Creation:

    This section provides an example of creating objects of the "Car" class and demonstrates how to access fields and methods of objects.

  4. Instance Variables:

    This section explains that each object has its own set of instance variables (fields), and changes to one object do not affect other objects.

Java Classes and Objects Explanation

1. Class: Blueprint or Template

Definition: A class serves as a blueprint or template for creating objects in Java. It defines the structure, attributes, and behaviors that objects of the class will possess.

Components:

A class consists of various components that collectively define its structure:

  • Fields (Data Members): Variables within a class that represent the attributes or properties of objects. Each object created from the class has its own set of values for these fields.
  • Methods (Member Functions): Functions within a class that define the behaviors or actions that objects can perform. Methods operate on the fields to achieve specific functionalities.
  • Constructors: Special methods used for initializing objects when they are created. Constructors have the same name as the class and are automatically called during object instantiation.
  • Modifiers: Keywords like public, private, and protected that control access to the members of a class.

Example Class:

The following code snippet represents a simple class named Car with fields, a constructor, and a method: 

        
        public class Car {
            // Fields
            String make;
            String model;
            int year;
        
            // Constructor
            public Car(String make, String model, int year) {
                this.make = make;
                this.model = model;
                this.year = year;
            }
        
            // Method
            public void startEngine() {
                System.out.println("Engine started for " + make + " " + model);
            }
        }

2. Object: Instance of a Class

Definition: An object is an instance of a class, created using the new keyword. It represents a real-world entity, possessing both state (fields) and behavior (methods).

Creating Objects:

Creating objects involves using the new keyword to instantiate instances of a class. Each object created has its own set of values for the fields, making it a unique instance.

Instance Variables:

Fields within an object are referred to as instance variables. Each object has its own instance of these variables, and changes to one object's fields do not affect the fields of other objects.

Example Object Creation and Usage:

The code snippet below demonstrates the creation and usage of objects of the Car class: 

        
        Car car1 = new Car("Toyota", "Camry", 2022);
        Car car2 = new Car("Honda", "Accord", 2021);
        
        System.out.println(car1.make);      // Output: Toyota
        car2.startEngine();                  // Output: Engine started for Honda Accord

3. Summary

Class and Object Relationship: A class defines the structure, and objects are instances created based on that structure. Classes provide a way to model and organize code, facilitating code reuse and maintainability.

Advantages of Classes and Objects:

Classes and objects offer several advantages in Java programming:

  • Code Reusability: Classes allow for the creation of multiple objects with shared behaviors and attributes. Code written in a class can be reused in various parts of a program.
  • Modularity: Classes promote modularity by encapsulating related functionalities. Each class can be developed, tested, and modified independently.

Java Object Class Methods

In Java, the Object class is a fundamental class that serves as the root of the class hierarchy. Every class in Java is a direct or indirect subclass of the Object class. It is part of the java.lang package, which is automatically imported in every Java program.

Methods in the Object Class:

The Object class provides several methods that are commonly used in Java classes. Here are some key methods:

  • toString() Returns a string representation of the object. This method is often overridden in custom classes to provide a meaningful string representation of the object.
  • equals(Object obj) Compares two objects for equality. The default implementation in the Object class compares the memory addresses of the two objects (reference equality). This method is often overridden to provide custom equality checks.
  • hashCode() Returns a hashcode value for the object. The default implementation returns the internal memory address of the object in integer form. This method is used in hash-based data structures like HashMap.
  • getClass() Returns the runtime class of the object. It is useful for obtaining information about the class of an object, especially in situations where polymorphism is involved.
  • clone() Creates and returns a copy of the object. To use this method, a class must implement the Cloneable interface and override the clone method to provide a proper cloning mechanism.
  • notify(), notifyAll(), and wait() These methods are used for inter-thread communication and synchronization. They are part of the mechanisms Java provides for managing concurrent programming.

Default Implementation:

The methods in the Object class have default implementations, and they can be overridden by any class that extends Object. Overriding these methods allows classes to define their own behavior for object comparison, string representation, and more.

Common Usage:

The Object class is commonly used when a method or a data structure needs to work with objects of any type. For example, when working with collections (like ArrayList or HashMap), objects of any type can be added, and the Object class is often used as the common type.

Example:

Here is a simple example demonstrating the use of the Object class: 

                
        public class ObjectExample {
            public static void main(String[] args) {
                // Creating objects of different types
                Object object1 = new Integer(42);
                Object object2 = "Hello, Java!";
                Object object3 = new Double(3.14);
        
                // Using methods of the Object class
                System.out.println("toString(): " + object1.toString());
                System.out.println("equals(): " + object1.equals(object2));
                System.out.println("hashCode(): " + object2.hashCode());
                System.out.println("getClass(): " + object3.getClass().getName());
            }
        }

In this example, objects of different types are assigned to variables of type Object, and then methods of the Object class are used on these objects.


Java Object Class Methods With Example

toString() Method:

The toString() method returns a string representation of the object. It is commonly overridden in classes to provide a meaningful representation of the object's state.

                
        public class ObjectToStringExample {
            public static void main(String[] args) {
                ObjectExample objectExample = new ObjectExample();
                System.out.println(objectExample.toString()); // Calls toString() implicitly
            }
        }
        
        class ObjectExample {
            @Override
            public String toString() {
                return "This is an ObjectExample instance.";
            }
        }

equals(Object obj) Method:

The equals() method is used to compare two objects for equality. It is commonly overridden in classes to provide custom equality comparison based on the object's properties.

                
        public class ObjectEqualsExample {
            public static void main(String[] args) {
                ObjectExample obj1 = new ObjectExample();
                ObjectExample obj2 = new ObjectExample();
        
                System.out.println(obj1.equals(obj2)); // Calls equals() implicitly
            }
        }
        
        class ObjectExample {
            @Override
            public boolean equals(Object obj) {
                if (this == obj) return true;
                if (obj == null || getClass() != obj.getClass()) return false;
                return true;
            }
        }

                System.out.println(objectExample.hashCode()); // Calls hashCode() implicitly
            }

                return 42;
            }
        }

clone() Method:

The clone() method creates and returns a copy of the object. It is commonly overridden in classes to provide a deep or shallow copy of the object.

                
        public class ObjectCloneExample {
            public static void main(String[] args) throws CloneNotSupportedException {
                ObjectExample original = new ObjectExample();
                ObjectExample clone = (ObjectExample) original.clone(); // Calls clone() explicitly
            }
        }
        
        class ObjectExample implements Cloneable {
            @Override
            protected Object clone() throws CloneNotSupportedException {
                return super.clone();
            }
        }

notify(), notifyAll(), and wait() Methods:

These methods are used for inter-thread communication and synchronization in Java. They are typically used in a multi-threaded environment to coordinate the execution of threads.

                
        public class ObjectThreadExample {
            public static void main(String[] args) throws InterruptedException {
                ObjectExample objectExample = new ObjectExample();
        
                // Thread 1
                Thread thread1 = new Thread(() -> {
                    synchronized (objectExample) {
                        try {
                            objectExample.wait();
                        } catch (InterruptedException e) {
                            e.printStackTrace();
                        }
                    }
                });
        
                // Thread 2
                Thread thread2 = new Thread(() -> {
                    synchronized (objectExample) {
                        objectExample.notify();
                    }
                });
        
                thread1.start();
                Thread.sleep(100);
                thread2.start();
            }
        }
        
        class ObjectExample {
            // Class definition
        }

Problem 1: Rectangle Class

The following code defines a Java class named Rectangle. This class has private fields for the length and width of a rectangle, a constructor to initialize these fields, and a method to calculate the area of the rectangle. 

        
        public class Rectangle {
            // Fields
            private double length;
            private double width;
        
            // Constructor
            public Rectangle(double length, double width) {
                this.length = length;
                this.width = width;
            }
        
            // Method to calculate area
            public double calculateArea() {
                return length * width;
            }
        }

Problem 3: BankAccount Class

The following code introduces a Java class called BankAccount. It includes private fields for the account number and balance, two constructors (default and parameterized), and methods for depositing, withdrawing, and displaying account information. 

        
        public class BankAccount {
            // Fields
            private String accountNumber;
            private double balance;
        
            // Default constructor
            public BankAccount() {
                this.accountNumber = "DefaultAccount";
                this.balance = 0.0;
            }
        
            // Parameterized constructor
            public BankAccount(double initialBalance) {
                this.accountNumber = "CustomAccount";
                this.balance = initialBalance;
            }
        
            // Method to deposit money
            public void deposit(double amount) {
                balance += amount;
            }
        
            // Method to withdraw money
            public void withdraw(double amount) {
                if (amount <= balance) {
                    balance -= amount;
                } else {
                    System.out.println("Insufficient funds!");
                }
            }
        
            // Method to display account information
            public void displayInfo() {
                System.out.println("Account Number: " + accountNumber + ", Balance: $" + balance);
            }
        }
